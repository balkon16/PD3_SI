# Definiujemy klas, ktra bdzie przetrzymywa wyniki oblicze po kadej epoce
class Stats(tflearn.callbacks.Callback):
    def __init__(self, examples=0):
        self.epoch_data = []
        self.data_size = examples
        self.last_time = 0
        self.start_time = 0
        
        self.min_report_secs = 1.

    def on_train_begin(self, training_state):
        self.start_time = time.time()
      
    def on_epoch_end(self, training_state):
        metrics = {
            'loss': training_state.loss_value, # loss (strata) dla danych treningowych
            'acc': training_state.acc_value,   # accuracy (dokadno) dla danych treningowych
            'val_loss': training_state.val_loss, # loss dla danych walidacyjnych
            'val_acc': training_state.val_acc, # accuracy dla danych walidacyjnych
            'epoch': training_state.epoch,
            'step': training_state.step,
            'iter': training_state.current_iter,
            'time': time.time() - self.start_time,
        }

        self.epoch_data.append(metrics)
        
    def on_batch_end(self, training_state, snapshot=False):
      cur_time = time.time()
      if cur_time - self.last_time < self.min_report_secs:
        return # nie spamujmy za czsto
      
      self.last_time = cur_time
      epoch = training_state.epoch
      step = training_state.step
      iter = training_state.current_iter
      print("Epoch %d, step (batch no.): %d -- acc: %.2f, loss %.2f -- iter %05d/%05d, training for: %.2fs" % (
          epoch, step, training_state.acc_value, training_state.loss_value,
          iter, self.data_size, cur_time-self.start_time))

\

Poniej jest przykad budowy sieci przy korzystaniu z tflearn. W zadaniu bdzie trzeba zmienia odpowiednie parametry.

Sie budujemy kolejnymi warstwami, gdzie funkcja tworzca nastpn warstw przyjmuje zmienn reprezentujc poprzedni. Dla wygody wynik przypisujemy na t sam zmienn, aby mc atwo dodawa dodatkowe warstwy poprzez wklejenie kodu w odpowiednie miejsce:

network = ... <- opis warstwy A \ network = conv_2d(network, ...) <- tworzymy now warstw B za warstw A i przypisujemy na t sam zmienn (nie potrzebujemy ju odwoywa si do warstwy A) \ network = fully_connected(network, ...) <- warstwa C za warstw B

W tym momencie zmienna network reprezentuje sie skadajc si z warstw A-B-C.

Uczenie¶

Uczenie sieci przebiega w fazach. Podajemy sieci przykady kolejno i stosujemy propagacj wstczn. W momencie, gdy przykady nam si skocz, po prostu zaczynamy od nowa. Kady taki obrt bdziemy nazywali epok. \ Przykadw nie bdziemy podawa sieci pojedynczo podajemy je w porcjach rozmiaru batch (wsad), co oznacza ile przykadw sie bdzie przetwarza jednoczenie.

Przydatne nam bd przede wszystkim funkcje:

conv_2d(network, #filtrw, #rozmiar, activation=f.aktywacji, regularizer="L2") - tworzy now warstw konwolucyjn posiadajc #filtrw filtrw (neuronw) o rozmiarze #rozmiar x #rozmiar oraz z funkcj aktywacji jak podana. Wartoci regularizer nie naley zmienia w tym zadaniu.

max_pool_2d(network, #rozmiar) - podprbkowanie danych co #rozmiar (zmniejsza wymiar obrazka razy #rozmiar)

fully_connected(network, #neuronw, activation=f.aktywacji) - warstwa pena z #neuronw neuronw i podan funkcj aktywacji.

Trenowanie sieci¶

Trenowanie sieci wywoujemy jak poniej (wszystkie 3 linijki bd potrzebne). Trenowanie moemy w dowolnym momencie przerwa klikajc na kwadrat zatrzymywania.

Metoda uczenia ma nastpujc sygnatur: model.fit({'input': X}, {'target': Y}, n_epoch=#epok, validation_set=({'input': testX}, {'target': testY}), show_metric=True, callbacks=[scores])

Uczy podan wyej sie, z danymi treningowymi z X i etykietami z Y przez #epok epok. Wyniki walidacji bd obliczane na zbiorze testowym testX z etykietami testY. Reszt zostawiamy bez zmian suy do wypisywania i rejestrowania statystyk.

Podczas uczenia bdziemy zbierali miary tego, jak sie radzi sobie z naszymi danymi:

loss - loss (strata) dla danych treningowych \ acc - accuracy (dokadno) dla danych treningowych \ val_loss - loss dla danych walidacyjnych \ val_acc - accuracy dla danych walidacyjnych \ epoch - numer epoki \ step - numer wsadu (batch) \ iter - liczba widzianych przykadw \ time - czas jaki upyn od pocztku uczenia \

Uwaga! kolejne wywoania poniszej komrki bd doucza sie aby uczy j od nowa musimy ponownie wykona komrk z definicj sieci.
